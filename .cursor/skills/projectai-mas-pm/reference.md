# Справочник: агенты и терминология ProjectAI

## Агенты по уровням и решаемым проблемам

### Стратегические проблемы

| Проблема | Агент | Назначение |
|----------|--------|------------|
| Размытые цели, слабая декомпозиция | Агент планирования | Декомпозиция целей на задачи с измеримыми показателями |
| Отсутствие риск-менеджмента | Агент риск-менеджмента | Анализ данных, мониторинг, выявление рисков, оценка вероятности, уведомления, корректирующие меры, эскалация |
| Слабая работа с Lessons Learned | Агент анализа Lessons Learned | Сбор данных о прошлых проектах, выявление повторяющихся ошибок, рекомендации, сравнение с историей, улучшение методологии |

### Процессные проблемы

| Проблема | Агент | Назначение |
|----------|--------|------------|
| Неясное распределение ролей | Агент управления ролями | Анализ структуры, компетенций, загрузки; рекомендации по назначению; матрица ролей и ответственности |
| Низкая прозрачность статусов | Агент мониторинга проектов | Сбор и обновление данных о статусах, зависимостях, сроках, загрузке; передача в BI; контроль актуальности |
| — | Агент уведомлений | Оповещения о критических изменениях |
| Зависимости между задачами/подразделениями | Агент координации зависимостей | Организационный уровень: межкомандная координация, бэклоги. Технический уровень: зависимости задач, обход блокировок, эскалация |
| Отсутствие приоритизации | Агент приоритизации задач | Ранжирование по целям, срокам, зависимостям, рискам; порядок выполнения; уведомления об изменении приоритетов |
| Управление сроками | Агент прогнозирования сроков | Прогноз задержек по истории и метрикам. Агент адаптации планов: пересчёт расписания, корректировки |
| Нет эскалации проблем | Агент эскалации проблем | Выявление блокировок, передача на вышестоящий уровень при превышении срока, уведомления, варианты решений |
| Управление изменениями | Агент моделирования изменений | Влияние изменений на сроки/ресурсы/зависимости, риски, сценарии внедрения, согласование, информирование |

### Операционные проблемы

| Проблема | Агент | Назначение |
|----------|--------|------------|
| Несогласованность процессов, разрозненные системы | Агент координации | Поиск данных (статусы, зависимости, риски из Jira, Confluence, ERP и т.д.); координация процессов, синхронизация |
| Неэффективное распределение ресурсов | Агент управления загрузкой ресурсов | Анализ распределения задач, прогноз перегрузок, рекомендации по перераспределению с учётом приоритетов и компетенций |
| Административная нагрузка, ручная отчётность | Агент автоматизации документации | Протоколы, поручения. Агент генерации отчётов: брифы, отчёты в реальном времени, KPI, визуализация. Агент уведомлений |

## Терминология

- **МАС** — мультиагентная система (multi-agent system, MAS).
- **Агент** — программный компонент с автономным поведением и чёткой ролью в рамках МАС.
- **Декомпозиция (целей)** — разбиение высокоуровневых целей на подцели и измеримые задачи.
- **Риск-менеджмент** — выявление, оценка и снижение рисков проекта.
- **Lessons Learned** — извлечённые уроки из завершённых проектов для предотвращения повторения ошибок.
- **Эскалация** — передача проблемы на более высокий уровень управления при невозможности решить в срок.
- **Координация зависимостей** — учёт и управление связями между задачами и между подразделениями.
