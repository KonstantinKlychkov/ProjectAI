# Cursor Integration

Использование фреймворка маппинга процессов PMI при настройке Cursor (Skills и Agents).

## Источники истины

- **framework/pmi-processes-mapping/** — домены, процессы, маппинг по доменам. Соответствие процессов и компонентов МАС будет определено после концепции МАС.
- **mas/requirements/business_requirements.md** — решаемые проблемы, функциональные требования к МАС. При необходимости работы по доменам и процессам подключать файлы из `framework/pmi-processes-mapping/` (например, domain-risk.md, reference/pmi-processes-reference.md).

## Субагенты / компоненты МАС

Субагенты и компоненты МАС будут определены в концепции МАС. Заготовки в .cursor/agents/ — опционально после её появления (например, по доменам на основе domain-*.md и раздела «Контекст для компонента МАС» в них).

## Иерархия и делегирование

Иерархия и правила делегирования относятся к архитектуре МАС и будут определены в концепции МАС. Источник истины — документ в зоне МАС: `mas/concept/` или `mas/architecture/` (после появления концепции). См. также [agent-hierarchy-and-delegation.md](agent-hierarchy-and-delegation.md) (ссылка на mas/).
